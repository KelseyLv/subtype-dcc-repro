# UCSC Xena → Subtype-DCC `*.fea`：预处理与特征选择说明

本文对应仓库脚本 `**repro/xena_gz_to_subtype_dcc_fea.py**`，便于在论文/开题/复现报告中描述 **「从 Xena 原始矩阵到模型输入」** 的预处理环节。实现以脚本为准；下列步骤顺序与代码一致。

---

## 1. 输入与输出约定

**输入（Xena）**：四个制表符分隔的 `.gz` 矩阵，行为 **特征**（基因 / miRNA / 探针 / GISTIC 基因），列为 **样本**（TCGA aliquot 等 ID）；**第一列**为特征索引列（列名因文件而异，如 `sample`、`Gene Symbol` 等），**不作为样本**参与交集。

**输出（Subtype-DCC）**：同一癌种目录下四个 **逗号分隔** 的 CSV：`rna.fea`、`miRNA.fea`、`meth.fea`、`CN.fea`。首列仍为特征索引，首行为样本 ID；**四个文件的列名集合与列顺序一致**。

脚本还会在同级目录写入 `**xena_to_fea_meta.txt`**，记录样本数与各模态最终行数及默认截断上限，便于审计。

---

## 2. 样本对齐（清洗核心）

1. **列名解析**：对每个 `.gz` 只读表头，取 **除第一列外** 的所有列名，视为候选样本 ID（避免把特征索引列误当作样本）。
2. **交集**：仅在 **四个矩阵同时出现** 的样本 ID 上取 **集合交集**，再按字典序排序，得到统一的列顺序 `sample_order`。未出现在全部四张表中的样本 **整例剔除**，保证多组学对齐。
3. **（可选）原发灶启发式过滤**：若使用 `--primary-tumor-only`，仅保留列名（经规范化后）以 `**01A`、`01B`、`01`** 结尾的 ID，作为 TCGA 原发肿瘤样本的常用启发式规则；非 TCGA 队列慎用。

后续所有矩阵只读取 **索引列 + `sample_order`** 中的列，减少 IO 与内存。

---

## 3. 数值读入与缺失处理

1. **类型**：各单元格经 `pandas.to_numeric(..., errors="coerce")` 转为数值，无法解析的记为 **NaN**。
2. **无穷大**：将 `±inf` 替换为 NaN，与缺失统一处理。
3. **缺失填补（按行）**：对每个特征（每一行），用该特征在 **全体保留样本上** 的 **中位数** 填补该行的 NaN（**行内中位数 imputation**）。这样在报告里可表述为：在固定样本集合上，对特征维缺失采用行内中位数填充，避免整列删除导致样本丢失。

> 说明：Xena 已发布的表达/甲基化矩阵通常已做过平台级处理；本脚本 **未再施加** log2(TPM+1) 等额外变换，以保持与「直接下载矩阵」的复现链一致。若需与某篇文献完全一致，应在文档中单独声明是否在 Xena 外增加了变换。

---

## 4. DNA 甲基化：特征选择与两阶段读入

TCGA 450K 甲基化矩阵行数约 **45 万级探针**，若一次性读入会占用大量内存，且直接作为深度学习输入维度过高。本复现管线采用 **无监督、按样本间方差排序** 的探针筛选，分 **两阶段** 完成。

### 4.1 目标

在全体样本列 `sample_order` 上，从所有探针中选出至多 `**K` 个**（默认 `K = 3139`，与仓库内部分已有癌种 `meth.fea` 行规模同量级，可通过 `--meth-max-probes` 修改），使选中的探针在样本维度上具有 **较大的跨样本方差**。

### 4.2 方差定义

对每一探针（矩阵的一行），在保留样本上的取值为 x_1,\ldots,x_n，使用 **总体方差**（脚本中 `numpy.nanvar(..., axis=1, ddof=0)`）：

- 对 NaN 使用 `nan` 方差语义，即仅基于非缺失观测计算；
- 若方差非有限值则视为 **0**，不参与提升排序。

该准则为 **完全无监督**：不利用标签、临床或生存信息，便于在方法章节归类为 **基于方差的探针压缩 / 高变探针子集**。

### 4.3 第一阶段（流式扫描）

- 以固定 `**chunksize`**（默认 25000 行）分块读取甲基化 `.gz`，每块仅含索引列与 `sample_order`。
- 对块内每一行计算上述行向方差。
- 维护一个大小至多为 `**K` 的小根堆**（按方差排序）：堆中始终保留当前已见探针中方差最大的 `K` 个；新探针方差若高于堆顶则替换。这样 **全程无需** 将整表载入内存。

第一阶段输出：**入选探针 ID 列表**（按方差从高到低排序，便于追溯）。

### 4.4 第二阶段（子集装载）

- 再次分块读同一甲基化文件，仅 **保留探针 ID 属于第一阶段列表** 的行，`concat` 后按第一阶段给出的 ID 顺序重排行索引。
- 对该子矩阵执行与第 3 节相同的 **行内中位数填补**。

甲基化模态 **不再** 在填补后做第二次「按方差截断行数」（RNA/miRNA/CN 仍有见第 5 节）；行数即 **至多 `K`**。

---

## 5. RNA、miRNA、拷贝数：高变特征截断（可选规模）

在样本对齐、数值化与中位数填补之后：

- **RNA**：按行计算与第 4.2 节相同的样本向方差，按方差从高到低排序，保留前 `**N_rna`** 个基因（默认 **3500**，`--rna-max-genes`）。
- **miRNA**：同理，保留前 `**N_mir`** 个（默认 **450**，`--mirna-max`）。
- **CN（GISTIC 阈值化基因）**：同理，保留前 `**N_cn`** 个（默认 **3500**，`--cn-max-genes`）。

与甲基化类似，该步骤为 **无监督方差筛选**，用于控制输入维度与与仓库既有癌种特征规模大致可比；**不是**差异甲基化/差异表达意义上的统计检验。

---

## 6. 默认超参与可复现记录


| 参数                  | 默认值  | 含义               |
| ------------------- | ---- | ---------------- |
| `--rna-max-genes`   | 3500 | RNA 保留的高变基因数上限   |
| `--mirna-max`       | 450  | miRNA 保留的高变特征数上限 |
| `--meth-max-probes` | 3139 | 甲基化保留的高方差探针数     |
| `--cn-max-genes`    | 3500 | CN 保留的高变基因数上限    |


建议在数据说明或附录中写明：**Xena 数据集版本、下载日期、四个文件名、是否使用 `--primary-tumor-only`、以及上表参数**。`xena_to_fea_meta.txt` 可作为运行快照引用。

---

## 7. 方法学表述提示（供答辩/论文用）

- **优点**：四组学 **严格样本交集**；缺失处理规则简单、可重复；甲基化在 **不加载全矩阵** 的前提下完成全局 top‑`K` 方差探针选择，适合工程复现。
- **局限**：方差筛选 **偏好高噪声或极端变异探针** 的可能存在，生物学解释需结合文献；若要与某篇仅用差异探针或预定义 probe set 的工作对比，应明确 **本管线与对方的特征空间不同**。
- **可扩展**：若希望甲基化用预定义列表，需改脚本或在外部先筛好探针再写入 `meth.fea`，并在文档中单列一条流程。
